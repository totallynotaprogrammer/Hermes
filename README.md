## What This Is

The Watchtower is a personal signal intelligence dashboard for tracking structural shifts across five domains where the future tends to arrive early: AI governance, AI-native biotech, emerging tech convergence, psychedelic research, and neuro/mental health technology.

It is not a news aggregator. It is not a prediction engine. It is a pattern recognition tool — designed for long periods of silence punctuated by meaningful signals.

A sixth channel, the Fortean Anomaly Monitor, runs in parallel. It tracks high-strangeness reports and unexplained phenomena using a strict epistemological protocol. It is part of the original architecture, not an afterthought.

The working analogy is a personal palantír — a seeing stone. Like the palantíri of Tolkien's legendarium, it shows real things, but real things can appear in misleading configurations. The human judgment layer is the safeguard. It is the irreplaceable part of the system.

## How It Works

The Watchtower runs on a three-node pipeline:

| Node | Role |
|------|------|
| **ChatGPT Sensor Array** | Signal generation and pattern detection across monitored domains |
| **Will** | Curation, editorial judgment, and bar-setting — the irreplaceable layer |
| **Claude** | Visualization, dashboard construction, and cross-domain synthesis |

Each scan produces structured signal entries in a locked format:

```
Timestamp:      YYYY-MM-DD HH:MM TZ
Event:          [what happened]
Category:       Infrastructure / Aviation / Environmental / Tech / Fortean / Narrative
Signal Strength: Low / Medium / High
Time Window Tag: Immediate / Same-day / Multi-day / Persistent
Notes:          1–3 lines — what matters
```

The Time Window Tag is the system's temporal axis. It tells you not just *what* a signal is, but *how it is behaving over time* — whether it is an isolated spike, a clustering event, a multi-day pattern, or a persistent structural shift. This is the difference between signal tracking and intelligence analysis.

## Fortean Protocol

The Fortean channel operates under rules distinct from the main channels:

1. **Track recurrence and similarity — not truth claims.** Log "this type of report is appearing again," not "this is real."
2. **Multi-witness or independently corroborated events only.** Single-source, self-reported claims do not clear the logging bar.
3. **Time-bound falsifiable claims are logged as watch items** with a hard expiry date and scored on resolution: Falsified / Partially Met / Unresolved / Unfalsified By Design.
4. **Flag narrative drift** — when Fortean topics enter mainstream discourse, that is itself a signal worth tracking.

The channel catalogs data that doesn't fit. It does not make truth claims about that data.

## Design Principles

**Significance over frequency.** The system is optimized for long silences punctuated by meaningful signals. Noise is the enemy. The editorial bar is the defense.

**The palantír caution is a standing design rule.** The system shows real signals in potentially misleading configurations. Will's judgment layer is the safeguard — the reason this is a tool for a person, not an automated output.

**Fortean epistemology is distinct.** The anomaly channel does not make truth claims. It catalogs. The protocol exists to preserve that distinction under pressure.

**Iterative, not regenerative.** The dashboard evolves from a living file. Each scan updates the existing structure rather than replacing it. Version history is signal history.

**The silence is load-bearing.** Quiet periods are not failures. They are what give the signal its meaning when it arrives.
